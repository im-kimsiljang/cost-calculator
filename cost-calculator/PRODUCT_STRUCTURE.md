# 식당 원가 계산기 — 메뉴 구성 & 사용자 플로우

> **서비스 컨셉**: 식당 사장님을 위한 메뉴 원가 관리 & 의사결정 도우미
> **핵심 가치**: "영수증 찍으면 자동으로 원가 분석하고, 뭘 해야 할지 알려주는 비서"
> **톤앤매너**: 부드럽고 중립적으로. "주의!", "경고!" 같은 센 표현 대신 정보 제공 느낌.

---

## 🎯 핵심 기능 (4가지)

1. **식자재 등록** — OCR로 영수증 찍어서 쉽게
2. **메뉴 구성** — 등록한 식자재로 레시피 만들기
3. **원가 분석** — 메뉴별 원가율·적정가 자동 계산
4. **어드바이스** — 원가율이 높을 때 조정 옵션 제안 (가격·양·대체재)

---

## 📊 판단 기준

### 1차 기준: 내 목표 원가율 (온보딩에서 설정)
사장님이 온보딩 때 직접 설정한 목표값. **강조하지 않고 은은하게** 표시.

### 2차 기준: 업종 평균 범위
업종 평균과 자연스럽게 비교. 참고 정보로만 제공.

**표시 방식 (부드럽게):**
- 🟢 좋아요 — 목표 또는 업종 평균 범위 내
- 🟡 살펴보세요 — 업종 평균을 약간 초과
- 🟠 높은 편이에요 — 업종 평균을 크게 초과

> ❌ "주의 필요", "관리 필요", "경고" 같은 센 표현은 쓰지 않음.
> ✅ 정보 제공 톤. 부담 없이 사장님이 스스로 판단하도록.

## 📱 메뉴 구성 (4탭)

```
┌─────────┬──────────┬─────────┬─────────┐
│ 🏠 홈   │ 🥗 식자재 │ 🍽 메뉴  │ ⚙️ 설정 │
└─────────┴──────────┴─────────┴─────────┘
```

### 🏠 홈 — "오늘 가게 상태 어떤가요?"
사장님이 앱을 켜면 **가장 먼저 보는 진단 화면**.

- **이번 달 상태 요약**
  - 전체 메뉴 평균 원가율
  - 업종 평균과 비교 (은은하게 참고 정보로)
  - 등록된 메뉴·식자재 개수
- **📍 살펴볼 메뉴 Top 3**
  - 원가율이 높은 메뉴 리스트 (센 표현 X)
  - 탭하면 해당 메뉴 상세로 이동
- **💡 오늘의 어드바이스 1건**
  - 부드러운 제안 ("돼지찌개 원가율 52%예요. 판매가 2,000원 정도 올리면 어떨까요?")
- **빠른 액션 버튼**
  - 📸 영수증 스캔
  - ➕ 메뉴 추가

---

### 🥗 식자재 — "재료를 넣자"
**OCR을 핵심 진입점으로** 배치. 수동 입력도 지원.

- **상단 CTA**: "📸 영수증 스캔" (Primary 버튼, 크게)
- **식자재 리스트**
  - 이름 / 구매가 / 용량 / 단위 / 단위당 가격
  - 탭하면 수정 가능
- **수동 추가 버튼**
- *(향후 확장)*
  - 단가 변동 이력
  - 가격 급등 알림
  - 구매처별 비교

---

### 🍽 메뉴 — "뭘 팔고 있나, 얼마 남나, 뭘 해보면 좋을까"
메뉴 등록 + 원가 분석 + 어드바이스를 한 곳에 통합.

#### 리스트 화면
- 기본 정렬: **원가율 높은 순** (살펴볼 메뉴가 위로)
- 각 메뉴 카드:
  - 메뉴명 / 판매가 / 원가율 (배지는 톤다운)
- **➕ 메뉴 추가 버튼**

#### 메뉴 상세 화면
- **레시피** (식자재 × 수량)
- **원가 분석 카드**
  - 원가 / 판매가 / 원가율 / 이익률
  - 내 목표(은은하게) · 업종 평균 범위 참고
- **💡 어드바이스 섹션** (원가율이 높을 때)
  - **가격 조정 옵션**: "2,500원 → 3,000원으로 올리면 원가율 42%→35%"
  - **양 조절 옵션**: "돼지고기 150g → 130g으로 줄이면 원가 -8%"
  - **대체 재료 옵션** *(향후)*: "한우 → 호주산 변경 시 원가 -15%"

---

### ⚙️ 설정
- 업종 / 목표 원가율 (온보딩에서 설정한 값, 여기서 수정 가능)
- 가게 정보 (이름, 주소, 대표메뉴)
- 공지사항 / 고객의견 / 약관

## 🔄 사용자 플로우

### 플로우 1: 신규 사장님 — 첫 사용
```
온보딩 (업종 선택 → 목표 원가율 설정 → 가게 정보)
    ↓
홈 화면 ("영수증 스캔부터 시작해요!" 안내)
    ↓
📸 영수증 스캔 → 식자재 자동 등록
    ↓
메뉴 추가 → 등록한 식자재로 레시피 구성
    ↓
원가율 확인 → 높으면 어드바이스 받고 조정
```

> 온보딩은 그대로 유지. 목표 원가율 설정도 유지하되 이후 화면에서는 **은은하게** 표시.

### 플로우 2: 기존 사장님 — 매일 쓰기
```
앱 실행 → 홈에서 "📍 살펴볼 메뉴" 또는 "💡 오늘의 어드바이스" 확인
    ↓
해당 메뉴 탭 → 어드바이스 보고 가격·양 조정 (부담 없이)
    ↓
영수증 스캔 (식자재 가격 업데이트) → 원가율 자동 재계산
```

### 플로우 3: 영수증 스캔 (핵심 플로우)
```
식자재 탭 → 📸 영수증 스캔 버튼
    ↓
카메라로 영수증 촬영
    ↓
CLOVA OCR 처리 (품목 / 수량 / 단가 추출)
    ↓
인식 결과 확인/수정 화면
    ↓
✅ 등록 → 식자재 리스트에 추가
    ↓
기존 메뉴의 원가율 자동 업데이트
    ↓
홈에 "식자재 N개 추가됨 · 영향받은 메뉴 M개" 알림
```

### 플로우 4: 어드바이스 받기 (차별화 포인트)
```
메뉴 상세 진입 → 원가율 48% (목표 35% · 업종 평균 30~38%)
    ↓
💡 어드바이스 섹션 (부드러운 톤):
   - "판매가를 8,500원 → 9,500원으로 올려보면 어떨까요?"
   - "또는 돼지고기 180g → 150g으로 줄이면 42%가 돼요"
   - "가장 비싼 재료: 돼지고기 (원가의 60%)"
    ↓
사장님 의사결정 (본인이 편하게):
   - 가격 조정 → 판매가 수정 버튼
   - 양 조절 → 레시피 수정 버튼
   - "이번엔 그대로 두기"
```

## 🚫 이번에 정리하는 것

### 운영비 관련 (전체 제거)
- ❌ **운영비 탭** (월세·인건비·공과금 등) — 복잡도만 늘려서 완전 제거
- ❌ 하단 4번째 탭 "운영통계" 삭제
- ❌ 홈의 "운영 원가율" 카드 제거
- ❌ 메뉴 상세의 "실질 마진" (운영비 포함 계산) 제거 → 순수 **메뉴 원가율**만 표시

### 목표/경고 표현 톤다운 (제거가 아니라 순화)
- ✅ **온보딩은 그대로 유지** (업종 선택·목표 원가율 설정 포함)
- ✅ 설정의 목표 원가율 항목도 유지
- 🔄 홈에서 "목표 대비 달성도"를 크게 강조하지 않고 **은은한 참고 정보**로
- ❌ "주의 필요", "관리 필요", "경고" 같은 센 표현 제거
- ✅ "살펴볼 메뉴", "살펴보세요", "높은 편이에요" 같은 부드러운 표현 사용

→ 핵심 질문: **"이 메뉴가 얼마나 남는지, 조정할 여지가 있는지"**를 편하게 안내.

---

## ✨ 차별화 포인트

1. **영수증 찍기 = 핵심 진입점**: 수기 입력 장벽 제거
2. **어드바이스 내재화**: 데이터만 보여주지 않고 "이렇게 해보면 어때요?" 제안
3. **부드러운 톤**: 압박 없이 편하게. 사장님이 스스로 판단
4. **홈은 진단, 메뉴는 처방**: 문제 발견부터 해결까지 1~2탭 안에서 완결

---

## 📐 화면 우선순위 (작업 순서)

1. **하단 탭 재구성** (4탭: 홈 / 식자재 / 메뉴 / 설정) ← 가장 먼저
2. **운영비 관련 코드 전체 제거**
3. **홈/메뉴 화면의 톤다운** (센 표현 → 부드러운 표현)
4. **홈 재설계** (살펴볼 메뉴 + 오늘의 어드바이스)
5. **식자재 탭 신설** + OCR 진입점
6. **메뉴 상세의 어드바이스 섹션**
